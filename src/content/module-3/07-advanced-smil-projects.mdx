# Advanced SMIL Projects

<Objective>
- Combine `<animate>`, `<animateTransform>`, and gradient-stop animation in one SVG
- Stagger entrance animations with `from`/`to` and `begin` offsets
- Drive interactive transitions via `begin="click"`
</Objective>

Four projects that push SMIL — morphing geometry, animated data viz, layered loaders, and gradient stop animation. Each is a recipe; rip out the colors and shapes for your own work.

## Project 1: Shape morph (circle → triangle → square → circle)

The morph rule from Module 4 applies: **same number of commands per keyframe**. Padding shorter shapes with redundant points keeps the morph smooth.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path fill="#7c5cff" stroke="#ffd43b" stroke-width="2" stroke-linejoin="round">
    <animate attributeName="d"
             values="M 50 20 C 70 20, 80 30, 80 50 C 80 70, 70 80, 50 80 C 30 80, 20 70, 20 50 C 20 30, 30 20, 50 20 Z;
                     M 50 20 L 80 70 L 80 70 L 50 80 L 20 70 L 20 70 L 50 20 Z;
                     M 20 20 L 80 20 L 80 80 L 80 80 L 20 80 L 20 80 L 20 20 Z;
                     M 50 20 C 70 20, 80 30, 80 50 C 80 70, 70 80, 50 80 C 30 80, 20 70, 20 50 C 20 30, 30 20, 50 20 Z"
             dur="6s" repeatCount="indefinite" calcMode="spline"
             keyTimes="0; 0.33; 0.66; 1"
             keySplines="0.4 0 0.2 1; 0.4 0 0.2 1; 0.4 0 0.2 1"/>
  </path>
</svg>`} caption="Each keyframe has 7 segments. Spline easing keeps it organic." />

## Project 2: Animated bar chart on enter

Bars rise from the baseline, staggered. `from`/`to` form is enough since each bar only goes one direction.

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <line x1="0" y1="90" x2="200" y2="90" stroke="rgba(255,255,255,0.1)" stroke-width="1"/>

  <rect x="20"  width="20" fill="#7c5cff" y="90" height="0">
    <animate attributeName="y"      from="90" to="30" dur="0.8s" begin="0s"   fill="freeze"/>
    <animate attributeName="height" from="0"  to="60" dur="0.8s" begin="0s"   fill="freeze"/>
  </rect>
  <rect x="60"  width="20" fill="#ffd43b" y="90" height="0">
    <animate attributeName="y"      from="90" to="50" dur="0.8s" begin="0.2s" fill="freeze"/>
    <animate attributeName="height" from="0"  to="40" dur="0.8s" begin="0.2s" fill="freeze"/>
  </rect>
  <rect x="100" width="20" fill="#3ddc97" y="90" height="0">
    <animate attributeName="y"      from="90" to="10" dur="0.8s" begin="0.4s" fill="freeze"/>
    <animate attributeName="height" from="0"  to="80" dur="0.8s" begin="0.4s" fill="freeze"/>
  </rect>
  <rect x="140" width="20" fill="#ff5d73" y="90" height="0">
    <animate attributeName="y"      from="90" to="60" dur="0.8s" begin="0.6s" fill="freeze"/>
    <animate attributeName="height" from="0"  to="30" dur="0.8s" begin="0.6s" fill="freeze"/>
  </rect>
</svg>`} caption="200ms stagger between bars. fill='freeze' keeps the final state." />

## Project 3: Multi-layer loader with animated gradient stops

Gradient stops are pure SMIL territory — CSS can't touch them.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <defs>
    <linearGradient id="rainbow" x1="0%" x2="100%">
      <stop offset="0%" stop-color="#7c5cff">
        <animate attributeName="stop-color"
                 values="#7c5cff;#3ddc97;#ffd43b;#ff5d73;#7c5cff"
                 dur="4s" repeatCount="indefinite"/>
      </stop>
      <stop offset="100%" stop-color="#ffd43b">
        <animate attributeName="stop-color"
                 values="#ffd43b;#ff5d73;#7c5cff;#3ddc97;#ffd43b"
                 dur="4s" repeatCount="indefinite"/>
      </stop>
    </linearGradient>
  </defs>

  <circle cx="50" cy="50" r="40" fill="none" stroke="rgba(255,255,255,0.08)" stroke-width="6"/>
  <circle cx="50" cy="50" r="40" fill="none" stroke="url(#rainbow)" stroke-width="6"
          stroke-linecap="round" stroke-dasharray="251.2" stroke-dashoffset="251.2">
    <animate attributeName="stroke-dashoffset" values="251.2;0;-251.2" dur="2s" repeatCount="indefinite"/>
    <animateTransform attributeName="transform" type="rotate"
                      from="0 50 50" to="360 50 50" dur="2s" repeatCount="indefinite"/>
  </circle>
</svg>`} caption="Rotation + dashoffset + animated gradient stops = a loader that's never the same shade twice." />

## Project 4: Click-to-toggle play / pause icon

`begin="click"` on the SVG itself, plus alternating `points` lists.

<LiveSVG code={`<svg viewBox="0 0 24 24" width="48" height="48" style="cursor:pointer">
  <polygon fill="#7c5cff"
           points="8,5 8,19 19,12">
    <animate attributeName="points"
             values="8,5 8,19 19,12; 7,5 7,19 11,19 11,5"
             dur="0.25s" begin="click" fill="freeze"
             keyTimes="0; 1" calcMode="spline" keySplines="0.2 0.8 0.2 1"/>
    <animate attributeName="points"
             values="7,5 7,19 11,19 11,5; 8,5 8,19 19,12"
             dur="0.25s" begin="click+0.5s" fill="freeze"
             keyTimes="0; 1" calcMode="spline" keySplines="0.2 0.8 0.2 1"/>
  </polygon>
</svg>`} caption="Click the triangle. Two chained animations toggle play ↔ pause via points morphing." />

## Key takeaways

1. **`<animate attributeName="d">`** unlocks shape morphing CSS can't do.
2. **Animated gradient stops** are exclusive SMIL territory.
3. **`from`/`to`** suffices for one-direction reveals; `values` for multi-step morphs.
4. **`fill="freeze"`** is mandatory for one-shot reveals.
5. **`begin="elementId.event"`** drives interactive sequences without JavaScript.
6. **Compose freely** — multiple `<animate>` elements on one shape, multiple shapes on one timeline.
