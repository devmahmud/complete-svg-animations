# Deep Dive into SMIL

<Objective>
- Pick `from`/`to` vs `values` for any animation shape
- Choose the correct `calcMode` (linear / paced / discrete / spline)
- Use `keyTimes` to pace keyframes asymmetrically
- Decide between `fill="freeze"` and `fill="remove"`
</Objective>

`<animate>` is the workhorse — every other SMIL element (`<animateTransform>`, `<animateMotion>`, `<set>`) is a specialized variant of the same idea: pick an attribute, list its values over time, and let the renderer interpolate.

## Anatomy

```html
<animate
  attributeName="cx"           <!-- which attribute -->
  values="50;100;50"           <!-- key values, semicolon-separated -->
  dur="2s"                     <!-- total duration -->
  begin="0s"                   <!-- start offset (or event) -->
  repeatCount="indefinite"     <!-- 1, n, or "indefinite" -->
  fill="freeze"                <!-- "remove" (default) or "freeze" final -->
  calcMode="linear"            <!-- linear, paced, discrete, spline -->
/>
```

## `values` vs `from`/`to`

There are two ways to specify keyframes:

```html
<!-- Two-value form: from start to end -->
<animate attributeName="r" from="10" to="40" dur="1s"/>

<!-- Multi-value form: arbitrary keyframes -->
<animate attributeName="r" values="10;40;15;30;10" dur="2s"/>
```

Use `from`/`to` for simple A→B. Use `values` for anything more complex — including ping-pong loops (start and end at the same value).

## `calcMode`: how SMIL interpolates

| Mode | Behavior |
| --- | --- |
| `linear` (default) | Even spacing between values |
| `paced` | Constant **visual** speed (great for path morphs) |
| `discrete` | Snap, no interpolation (good for fill cycling) |
| `spline` | Cubic Bézier easing per segment (next lesson) |

## `keyTimes`: pacing keyframes manually

`values` divides the duration evenly by default. `keyTimes` overrides that — each entry is a number 0–1 marking when the corresponding `values` entry happens.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <circle cx="50" cy="50" r="20" fill="#7c5cff">
    <!-- spend 80% of the time small, snap big in the last 20% -->
    <animate attributeName="r"
             values="20;20;50;20"
             keyTimes="0; 0.8; 0.9; 1"
             dur="2s" repeatCount="indefinite"/>
  </circle>
</svg>`} caption="Anticipation + payoff: hold small, then quick burst." />

## `repeatCount` and `repeatDur`

- `repeatCount="3"` — play three times.
- `repeatCount="indefinite"` — loop forever.
- `repeatDur="6s"` — repeat for a total of 6 seconds (regardless of how many cycles fit).

## `fill="freeze"` vs `fill="remove"`

When the animation ends, what happens to the attribute?

- **`remove`** (default) — snap back to the original value.
- **`freeze`** — keep the final value.

For one-shot reveals (path drawing, fade-in), always use `fill="freeze"` or the animation will reset on completion.

<Callout type="warn" title="Beware the `fill` collision">
  The <code>fill</code> attribute means TWO things: paint color (on shapes) and end-state (on animations). They never conflict because they're on different elements, but the dual meaning trips up everyone once.
</Callout>
