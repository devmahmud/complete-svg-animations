# The Seven Primitive Shapes

<Objective>
- Identify the seven primitive shapes and what each is good at
- Pick the right primitive for a given drawing task
- Read each shape's attributes well enough to modify them on sight
</Objective>

SVG has **seven primitive shapes** that form the foundation of every graphic. Each lives inside an `<svg>` viewBox and is positioned in user units.

## 1. Rectangle (`<rect>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <rect x="20" y="20" width="60" height="60" rx="8" fill="#7c5cff"/>
</svg>`} />

**Attributes:** `x`, `y` (top-left), `width`, `height`, `fill`, `stroke`, `stroke-width`, `rx`/`ry` for rounded corners.

## 2. Circle (`<circle>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <circle cx="50" cy="50" r="30" fill="#ffd43b" stroke="#7c5cff" stroke-width="3"/>
</svg>`} />

**Attributes:** `cx`, `cy` (center), `r` (radius).

## 3. Ellipse (`<ellipse>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <ellipse cx="50" cy="50" rx="40" ry="20" fill="none" stroke="#3ddc97" stroke-width="3"/>
</svg>`} />

**Attributes:** `cx`, `cy`, `rx` (horizontal radius), `ry` (vertical radius).

## 4. Line (`<line>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <line x1="20" y1="20" x2="80" y2="80" stroke="#ede9de" stroke-width="3" stroke-linecap="round"/>
</svg>`} />

**Attributes:** `x1`, `y1`, `x2`, `y2`. A line has no fill — only `stroke`.

## 5. Polyline (`<polyline>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <polyline points="20,80 40,40 60,60 80,20" fill="none" stroke="#7c5cff" stroke-width="3" stroke-linejoin="round"/>
</svg>`} />

**Attributes:** `points` is a list of `x,y` pairs. Polyline does **not** auto-close.

## 6. Polygon (`<polygon>`)

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <polygon points="50,15 90,80 10,80" fill="rgba(124,92,255,0.15)" stroke="#7c5cff" stroke-width="3" stroke-linejoin="round"/>
</svg>`} />

Same as polyline, but the last point auto-connects to the first.

## 7. Path (`<path>`)

The most powerful primitive — every other shape can be expressed as a path.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path d="M 20 20 L 80 20 L 80 80 L 20 80 Z" fill="none" stroke="#ffd43b" stroke-width="3"/>
</svg>`} caption="Try replacing 'L 80 20' with 'Q 50 -10 80 20' to curve the top edge." />

**Attribute:** `d` is the path's _command_ string (`M` move, `L` line, `Q` quadratic, `C` cubic, `Z` close…). We'll cover paths in depth in Module 4.

  <!-- A polygon needs 8 points for an octagon -->
  <polygon points="" fill="" stroke="" stroke-width=""/>
</svg>`} caption="Hint: an octagon's vertices live on a circle of radius ~38, evenly spaced 45° apart. Approximate to integers." />

<Callout type="tip" title="Mental model">
  Once you internalize that **every shape is a path**, attribute names start to make sense — they're just shorthand for path data.
</Callout>
