# Styling Attributes

<Objective>
- Apply `fill`, `stroke`, opacity, and color formats fluently
- Pick the right `stroke-linecap` and `stroke-linejoin` for the look you want
- Use `currentColor` so SVGs inherit color from their CSS context
</Objective>

SVG paint is controlled by attributes (or matching CSS properties). The four you'll reach for daily: `fill`, `stroke`, `stroke-width`, and the various `stroke-*` modifiers.

## Fill and stroke together

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <rect x="20" y="20" width="60" height="60"
        fill="#7c5cff"
        fill-opacity="0.6"
        stroke="#ffd43b"
        stroke-width="4"
        stroke-opacity="1"/>
</svg>`} caption="Drop fill-opacity to 0 to expose the stroke as an outline." />

| Attribute | Effect |
| --- | --- |
| `fill` | Interior color |
| `fill-opacity` | 0–1, fill alpha only |
| `stroke` | Outline color |
| `stroke-width` | Outline thickness |
| `stroke-opacity` | 0–1, outline alpha only |
| `opacity` | Combined alpha for fill + stroke |

## Color formats

SVG accepts every CSS color format you know:

```css
fill="red"                /* named */
fill="#ff5d73"            /* hex */
fill="rgb(255 93 115)"    /* rgb */
fill="rgb(255 93 115 / 0.5)"  /* with alpha */
fill="currentColor"       /* inherits the CSS `color` property */
```

`currentColor` is the secret weapon for icon systems — set `color` once on a parent and every child SVG inherits.

## Dashed lines

<LiveSVG code={`<svg viewBox="0 0 200 60">
  <line x1="10" y1="30" x2="190" y2="30"
        stroke="#3ddc97" stroke-width="3" stroke-dasharray="8 4"/>
</svg>`} caption='stroke-dasharray="8 4" → 8 units on, 4 off, repeat.' />

`stroke-dasharray` accepts a list — `"4 2 8 2"` produces a four-step pattern. We'll use this heavily for drawing animations in Module 4.

<Callout type="tip" title="Attributes vs CSS">
  Most paint attributes (<code>fill</code>, <code>stroke</code>, <code>opacity</code>) can be set in CSS too. Attributes have lower specificity than any CSS rule, which is exactly what you want — set defaults in markup, override in stylesheets.
</Callout>
