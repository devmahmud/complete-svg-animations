# Introduction to SVG Animations

<Objective>
- Pick the right animation system for the job (CSS transitions, keyframes, transforms, SMIL)
- Recognize what each system does well and where it falls short
- Build the mental model: state-changes vs ambient loops
</Objective>

You can draw shapes — now make them move. SVGs accept four animation systems, and you'll mix them depending on what each one does best:

| System | Strengths | Limits |
| --- | --- | --- |
| **CSS Transitions** | State changes (hover, focus, toggle) | Two states only |
| **CSS Keyframes** | Looping ambient motion | Some attributes are off-limits |
| **CSS Transforms** | GPU-accelerated movement, scale, rotation | Doesn't change geometry |
| **SMIL** | Animate any SVG attribute, motion paths, morphing | Not in Internet Explorer |

This module covers the CSS-based approaches. Module 3 introduces SMIL.

## Why animate at all?

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <circle cx="100" cy="50" r="20" fill="#7c5cff" class="bell"/>
</svg>

<style>
.bell {
  transform-origin: 100px 50px;
  animation: ring 2s ease-in-out infinite;
}
@keyframes ring {
  0%, 50%, 100% { transform: rotate(0); }
  60% { transform: rotate(15deg); }
  70% { transform: rotate(-12deg); }
  80% { transform: rotate(8deg); }
  90% { transform: rotate(-5deg); }
}
</style>`} caption="Five lines of CSS, the difference between a static dot and 'something is happening here.'" />

A static screenshot communicates **state**. Motion communicates **change**, **causality**, and **personality**. Use animation to:

- **Guide attention** — animate the thing the user should look at next.
- **Confirm action** — a button that springs slightly when clicked feels alive.
- **Show relationships** — when one element morphs into another, the user understands they're connected.
- **Signal status** — a pulsing dot says "loading" without using a single word.

<Callout type="warn" title="Restraint">
  Every animation has a perceptual cost. If three things move at once, the user can track none of them. Animate one focal element at a time.
</Callout>
