# Timing Functions

<Objective>
- Pick the right easing keyword for the kind of motion you want
- Author custom curves with `cubic-bezier`
- Use `steps()` for discrete, ticking motion
</Objective>

A timing function describes how an animation's progress maps from "0% done" to "100% done" — linear, eased, bouncy, elastic. The same duration with different easing **looks** completely different, even though the start and end states are identical.

| Keyword | Curve | When to use |
| --- | --- | --- |
| `linear` | constant speed | mechanical motion (gauges, timers) |
| `ease` | slight ease-in/out (default) | most things |
| `ease-in` | slow start, abrupt end | exits, things leaving the screen |
| `ease-out` | abrupt start, slow end | entries, things arriving |
| `ease-in-out` | symmetric ease | back-and-forth loops |

## Custom curves with `cubic-bezier`

`cubic-bezier(x1, y1, x2, y2)` defines a Bézier curve from (0,0) to (1,1). The two control points (x1, y1) and (x2, y2) shape the speed.

  <g class="row"><circle cx="20" cy="160" r="10" fill="#3ddc97" class="b elastic"/></g>

  <text x="40" y="44"  fill="#b8bfd0" font-size="11">smooth (0.25, 0.46, 0.45, 0.94)</text>
  <text x="40" y="104" fill="#b8bfd0" font-size="11">bounce (0.68, -0.55, 0.265, 1.55)</text>
  <text x="40" y="164" fill="#b8bfd0" font-size="11">elastic (0.175, 0.885, 0.32, 1.275)</text>
</svg>

<Callout type="tip" title="Visual editor">
  Don't memorize Bézier numbers. Use <a href="https://cubic-bezier.com">cubic-bezier.com</a> or DevTools' inline curve editor to draw the feel you want.
</Callout>

## Steps: discrete jumps

`steps(n)` divides the timeline into `n` equal jumps — perfect for sprite animation, typewriter effects, or "ticking" indicators.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <line x1="50" y1="50" x2="50" y2="20" stroke="#ffd43b" stroke-width="3" stroke-linecap="round" class="hand"/>
  <circle cx="50" cy="50" r="3" fill="#ffd43b"/>
</svg>

<style>
.hand {
  transform-origin: 50px 50px;
  animation: tick 12s steps(12) infinite;
}
@keyframes tick { to { transform: rotate(360deg); } }
</style>`} caption="Clock hand ticks 12 discrete times per loop." />

## Picking the right function

- **Match the metaphor.** Real objects have inertia → use eased curves. Mechanical things → linear.
- **Direction matters.** Things entering decelerate (`ease-out`); things leaving accelerate (`ease-in`).
- **Duration matters.** Easing is more visible at longer durations. For 100ms hover effects, linear vs ease is barely perceptible.
- **Overshoot adds personality.** A subtle bounce on a button click signals "I noticed you" without being annoying.
