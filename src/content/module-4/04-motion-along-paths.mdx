# Motion Along Paths

<Objective>
- Use `<animateMotion>` to move any element along a path
- Reuse a path across animations with `<mpath href="#id">`
- Apply `rotate="auto"` to align an element with the path's tangent
</Objective>

`<animateMotion>` moves any element along a path's trajectory. The element's own coordinates become offsets relative to the path — set them to (0, 0) so it sits **on** the path.

## Smallest example

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <!-- The track (visible) -->
  <path id="track" d="M 20 50 Q 100 -10 180 50"
        fill="none" stroke="rgba(255,255,255,0.2)" stroke-width="2" stroke-dasharray="3 3"/>

  <!-- The traveler -->
  <circle r="6" fill="#7c5cff">
    <animateMotion dur="3s" repeatCount="indefinite" path="M 20 50 Q 100 -10 180 50"/>
  </circle>
</svg>`} caption="The path attribute is repeated on the animateMotion. Or use mpath (next demo)." />

## Reuse the path with `<mpath>`

Cleaner — define the path once, reference it:

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <path id="track2" d="M 20 50 C 60 -10 140 110 180 50"
        fill="none" stroke="rgba(255,255,255,0.2)" stroke-width="2" stroke-dasharray="3 3"/>

  <circle r="6" fill="#ffd43b">
    <animateMotion dur="3s" repeatCount="indefinite">
      <mpath href="#track2"/>
    </animateMotion>
  </circle>
</svg>`} caption="No more duplicated path string." />

## Auto-rotate to face the direction of travel

`rotate="auto"` orients the element along the tangent of the path — perfect for arrows, ships, anything pointed.

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <path id="track3" d="M 20 50 C 60 0 140 100 180 50"
        fill="none" stroke="rgba(255,255,255,0.2)" stroke-width="2" stroke-dasharray="3 3"/>

  <g>
    <!-- An arrow pointing right; pivot at its tail -->
    <polygon points="-8,-4 -8,4 8,0" fill="#7c5cff"/>
    <animateMotion dur="3s" repeatCount="indefinite" rotate="auto">
      <mpath href="#track3"/>
    </animateMotion>
  </g>
</svg>`} caption="Try rotate='auto-reverse' to flip the orientation." />

## Caravan: stagger with `begin`

<LiveSVG code={`<svg viewBox="0 0 200 100">
  <path id="track4" d="M 20 50 Q 100 0 180 50" fill="none"
        stroke="rgba(255,255,255,0.2)" stroke-width="2" stroke-dasharray="3 3"/>
  <circle r="5" fill="#ff5d73"><animateMotion dur="3s" repeatCount="indefinite" begin="0s"><mpath href="#track4"/></animateMotion></circle>
  <circle r="5" fill="#7c5cff"><animateMotion dur="3s" repeatCount="indefinite" begin="0.5s"><mpath href="#track4"/></animateMotion></circle>
  <circle r="5" fill="#ffd43b"><animateMotion dur="3s" repeatCount="indefinite" begin="1s"><mpath href="#track4"/></animateMotion></circle>
</svg>`} caption="Same path, three offset start times." />

## CSS alternative: `offset-path`

Modern CSS supports the same effect via the `offset-path` property:

```css
.traveler {
  offset-path: path('M 20 50 Q 100 0 180 50');
  animation: glide 3s linear infinite;
}
@keyframes glide {
  to { offset-distance: 100%; }
}
```

The CSS approach has better DOM ergonomics; SMIL has wider browser support for complex synchronization.

<Callout type="warn" title="The element's own x/y matters">
  If you put <code>cx="50"</code> on the circle and then animate it along a path, every point on the path is offset by (50, 0). For most cases set <code>cx</code>/<code>cy</code> to 0.
</Callout>
