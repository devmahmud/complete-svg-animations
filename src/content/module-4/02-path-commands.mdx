# Path Commands

<Objective>
- Use M, L, H, V, Q, C, A, and Z confidently
- Distinguish absolute (uppercase) from relative (lowercase) commands
- Author a path from scratch given a target shape
</Objective>

Every path command is **one letter** followed by coordinates. **Capital letters use absolute coordinates**, lowercase use **relative** (delta from the previous point).

## The full command set

| Command | Meaning |
| --- | --- |
| `M` / `m` | Move to (no line drawn) |
| `L` / `l` | Line to |
| `H` / `h` | Horizontal line |
| `V` / `v` | Vertical line |
| `Q` / `q` | Quadratic Bézier (1 control point) |
| `T` / `t` | Smooth quadratic (mirror previous control) |
| `C` / `c` | Cubic Bézier (2 control points) |
| `S` / `s` | Smooth cubic (mirror previous control) |
| `A` / `a` | Elliptical arc |
| `Z` / `z` | Close path |

## Lines: M, L, H, V

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <!-- Square via H and V -->
  <path d="M 20 20 H 80 V 80 H 20 Z" fill="none" stroke="#7c5cff" stroke-width="3"/>
</svg>`} caption="H and V take only one number — they keep the other coordinate constant." />

## Absolute vs relative

The same square, written with relative commands:

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path d="M 20 20 h 60 v 60 h -60 z" fill="none" stroke="#ffd43b" stroke-width="3"/>
</svg>`} caption="Lowercase = delta. Easier to refactor; harder to read at first." />

## Quadratic curves: Q

`Q cx cy x y` — one control point, one end point. The curve bends toward the control.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path d="M 10 80 Q 50 0 90 80" fill="none" stroke="#7c5cff" stroke-width="3"/>
  <!-- Visualize the control -->
  <line x1="10" y1="80" x2="50" y2="0" stroke="#3ddc97" stroke-dasharray="2 2" stroke-width="1" opacity="0.6"/>
  <line x1="50" y1="0" x2="90" y2="80" stroke="#3ddc97" stroke-dasharray="2 2" stroke-width="1" opacity="0.6"/>
  <circle cx="50" cy="0" r="2" fill="#3ddc97"/>
</svg>`} caption="Green dot is the control point. Move it down and the curve flattens." />

## Cubic curves: C

`C c1x c1y c2x c2y x y` — two control points. Each anchors a tangent.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path d="M 10 80 C 30 10, 70 10, 90 80" fill="none" stroke="#7c5cff" stroke-width="3"/>
  <line x1="10" y1="80" x2="30" y2="10" stroke="#3ddc97" stroke-dasharray="2 2" stroke-width="1" opacity="0.6"/>
  <line x1="90" y1="80" x2="70" y2="10" stroke="#3ddc97" stroke-dasharray="2 2" stroke-width="1" opacity="0.6"/>
  <circle cx="30" cy="10" r="2" fill="#3ddc97"/>
  <circle cx="70" cy="10" r="2" fill="#3ddc97"/>
</svg>`} caption="Cubic = more shape control. Drag a control x-coord and feel the curve respond." />

## Arcs: A

`A rx ry x-rotation large-arc sweep x y` — the seven scariest numbers in SVG.

<LiveSVG code={`<svg viewBox="0 0 100 100">
  <path d="M 20 50 A 30 20 0 0 1 80 50" fill="none" stroke="#ffd43b" stroke-width="3"/>
</svg>`} caption='rx=30, ry=20, no rotation, small arc, clockwise sweep. Flip the 4th flag to 1 to take the long way around.' />

The `large-arc` and `sweep` flags pick which of the four possible arcs you mean — try toggling them between 0 and 1.

## Close: Z

`Z` (or `z`) draws a straight line back to the most recent `M`. It's how you make filled shapes — without it, fills can produce surprising results.

## A meaningful example

<LiveSVG code={`<svg viewBox="0 0 200 200">
  <path d="M 20 100
           Q 50 20 100 100
           T 180 100
           L 180 150
           Q 100 180 20 150
           Z"
        fill="rgba(124,92,255,0.18)"
        stroke="#7c5cff"
        stroke-width="2"
        stroke-linejoin="round"/>
</svg>`} caption="A wave with a curved baseline. Five commands, infinite variations." />

<Callout type="tip" title="Read paths like sentences">
  Each command is a verb. Coordinates are objects. <code>M 0 0 L 10 10 Z</code> = "move to origin, draw line to (10,10), close." Once that clicks, every SVG file becomes legible.
</Callout>
